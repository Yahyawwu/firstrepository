Exercise 1 (25 points)¶

The New York Times (NYT) APIs offer a rare free opportunity to analyse data since 1851 digitally (the year of the first issue of the paper) and thereby a multitude of historical events or also topics over time. You can e.g. use the "Article Search API" to search through all articles without obtaining their texts explicitly. Alternatively, you could use the "Archive API" which allows to download all article headlines and lead paragraphs/snippets for a given month (and through iteration potentially for a range of months or years). While this does not allow to obtain the full articles, it still allows to download a lot of textual data and thereby to do own textual analysis. There are also a range of other NYT APIs which you can find here if these are of greater interest to you. If you click on a specific API, you also find detailed documentation about its endpoint format etc.

Think about a topic that you are interested in and use the NYT APIs to develop and present an initial analysis of the topic. A good starting point can be to think about whether content in the NYT APIs could be useful for your PhD thesis, or other research projects. You could then use this exercise as a starting point and explore the NYT APIs. A macroeconomist could e.g. track the word share of "inflation" over some years either with the Article search API or the Archive API (in this case e.g. with pandas or sklearn and only based on headlines and/or lead paragraphs) and illustrate some discussions of inflation through word clouds; an accounting researcher could e.g. look up specific historical newspaper discussions of accounting standards or reporting scandals; a finance researcher historical stock market crashes or bank runs; etc. The analysis can be based on basic quantitative text analysis, visualisations e.g. of keyword frequencies and time series, word clouds, written discussion, etc. Obtain the data from the APIs with requests and feel free to use any Python package which is helpful for the analysis.

Hint: The API notebook from the lecture can be a good starting point for this exercise. If you plan to use the Archive API to do the text analysis in Python, note that the main available text is contained in the headline, abstract, lead_paragraph, and/or snippet columns (depending on which month/year you download some of these will be empty or the same). Also note that a string for the headline can be found under the key main in the headline column of the data frame created in the lecture (i.e. a row in that column contains a dictionary).

def obtain_monthly_counts(search_term, key, start_year, end_year):
    
    params = {"q": search_term,
              "api-key": key}
    
    all_counts = []
    
    for year in range(start_year, end_year+1):
        
            for month in range(1,13):
                
                if month<10:
                    month = "0"+str(month)

                params["begin_date"] = f"{year}{month}01"
                params["end"] = f"{year}{month}31"

                r = requests.get("https://api.nytimes.com/svc/search/v2/articlesearch.json", params=params)
                data = r.json()
                count = data["response"]["meta"]["hits"]

                # Wait 7 seconds between requests to hace some buffer as only 10 requests
                # per minute are allowed
                time.sleep(7)

                all_counts.append(count)
                
    counts_series = pd.Series(all_counts)
    counts_series.index = pd.date_range(pd.to_datetime(f"{start_year}0101"), periods=(end_year-start_year+1)*12, freq='M')
            
    return counts_series
